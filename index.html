802.1X + RADIUS
Authentication in Wireless Networks

A slide-style technical overview of how enterprise Wi-Fi authenticates users with identities, not shared passwords.

Key idea: Per-user authentication (WPA2/3-Enterprise) instead of a shared PSK.

Quick comparison

Home Wi-Fi

WPA2-PSK / WPA3-SAE

  • • One shared password
  • • Hard to track individuals
  • • Password gets shared/leaked

Enterprise Wi-Fi

WPA2/3-Enterprise (802.1X)

  • • Per-user credentials
  • • Central policy + logging
  • • Stronger control & auditing
Scope: Defensive learning only. No attack steps.

What is 802.1X?

IEEE 802.1X is a standard for port-based network access control. It authenticates a user/device before granting network access.

Used in

  • • WPA2-Enterprise
  • • WPA3-Enterprise
  • • Wired 802.1X (switch ports)

Replaces

  • • Shared Wi-Fi passwords (PSK)
  • • “Everyone knows the same key”
  • • Weak accountability

Core concept: the AP acts as a gatekeeper; the identity check happens on a central server.

What is RADIUS?

RADIUS (Remote Authentication Dial-In User Service) is an AAA protocol used for: Authentication, Authorization, and Accounting.

Why it matters for Wi-Fi

  • • Central place to validate users (Active Directory / LDAP / local DB)
  • • Enforces policies (who can join, which VLAN, which role)
  • • Logs every login attempt for auditing

Common servers: FreeRADIUS, Microsoft NPS, Aruba ClearPass, Cisco ISE.

The 3 key components

Supplicant

Client device

Laptop/phone that requests access and runs EAP authentication.

Authenticator

Access Point (AP)

Controls access and forwards authentication messages to RADIUS.

Authentication Server

RADIUS

Validates credentials and returns an “accept/deny” decision + policy.

Key point: The AP does not “know” the password — it asks the RADIUS server to verify identity.

How it works (EAP flow)

Step-by-step

  1. 1) Client connects to SSID
  2. 2) AP blocks network access (pre-auth)
  3. 3) EAP identity exchange starts
  4. 4) AP forwards EAP to RADIUS
  5. 5) RADIUS validates user (AD/LDAP)
  6. 6) Access-Accept + policy returned
  7. 7) Client gets network access

What gets enforced

  • • Role-based access (staff vs guest)
  • • VLAN assignment
  • • Time-based policies
  • • MFA / certificate requirements
  • • Logging (accounting)
EAP methods include: PEAP, EAP-TLS, TTLS (varies by organization).

Why it’s more secure than PSK

Identity-based access

Every user/device has its own credentials (or certificate). No shared password to leak.

Stronger auditing

RADIUS logs who connected, when, and from which device — useful for incident response.

Dynamic session keys

Enterprise authentication negotiates per-session encryption material, improving isolation.

Central policy control

One place to enforce posture, roles, VLANs, and revoke access instantly.

Where you see it in real life

802.1X is standard in enterprise environments where access must be controlled and audited:

Universities

Campus Wi-Fi with personal accounts (e.g., eduroam style setups).

Hospitals

Segmentation by role/device + strong audit logging.

Enterprises

Central identity, rapid revocation, policy compliance.

Government

Higher assurance access controls and accountability.

Summary

  • • 802.1X enables identity-based Wi-Fi access.
  • • The AP is the Authenticator, not the password keeper.
  • • RADIUS provides centralized AAA and logging.
  • • Enterprise Wi-Fi improves control, segmentation, and auditability.
